# 🧩 二、Agent框架（必问）

## 🎯 面试官想听什么

- 不是“用过LangChain”
- 而是：**你能不能自己搭一个Agent系统**

---

## 💡 标准回答结构

### 1️⃣ Agent系统分层（一定要讲）

```
1. LLM层（推理）
2. Orchestration层（Agent编排）
3. Tool层（外部能力）
4. Memory层（短期/长期）
5. Observability层（监控）
```

👉 面试话术：

> 我一般不会直接依赖框架，而是会抽象成5层架构，这样可以方便替换模型和工具。

---

### 2️⃣ LangGraph / AutoGen 怎么用（实战表达）

👉 不要说“我用过”，要说：

> 我用 LangGraph 主要是用它的 **State + DAG能力** 来做多Agent编排，例如：
>
> - 一个负责检索
> - 一个负责总结
> - 一个负责调用工具
>
> 同时我会在外层加一层 **调度控制（Golang）**，解决：
>
> - 超时控制
> - 重试
> - 降级

---

### 3️⃣ 从0搭Agent（高分点🔥）

👉 标准架构：

```
User Query
   ↓
Intent Router（小模型）
   ↓
Agent Planner
   ↓
Tool Executor（带重试）
   ↓
LLM Response
```

👉 面试话术：

> 我通常不会让LLM直接决定一切，而是：
>
> - 用小模型做意图分类
> - 再进入不同Agent flow
>   这样可以降低成本并提高稳定性

---

# 📚 三、企业级RAG（核心重点🔥🔥🔥）

## ❌ 错误回答（很多人会说）

- “我用向量数据库做检索”

👉 直接挂

---

## ✅ 正确回答结构（分4层）

### 1️⃣ 分片策略（Chunking）

👉 面试话术：

> 我不会简单按固定长度切chunk，而是：
>
> - 语义切分（按段落/标题）
> - 保持上下文完整性
> - 控制chunk size（避免embedding失真）

---

### 2️⃣ 多路召回（关键🔥）

```
向量检索（语义）
+ BM25（关键词）
+ SQL / Graph（结构化）
```

👉 面试话术：

> 线上RAG一定是多路召回，否则召回率不够

---

### 3️⃣ Rerank（区分高手）

👉 关键词：

- cross-encoder
- bge-reranker
- cohere rerank

👉 面试话术：

> 我一般会在召回后做rerank，把top20压缩到top5，提高准确率

---

### 4️⃣ 上下文融合（很多人不会🔥）

👉 方法：

- sliding window
- context compression
- 去重

👉 面试话术：

> 我会控制上下文token预算，比如：
>
> - 检索内容不超过40%
> - 保留对话历史
> - 避免上下文污染

---

# 🔧 四、工具调用可靠性（面试高频🔥）

## 标准设计（直接说这套）

```
Tool Layer:
- 参数校验（JSON Schema）
- Timeout控制
- 重试机制（指数退避）
- 幂等设计
- fallback策略
```

---

👉 面试话术：

> 我不会让LLM直接调用工具，而是：
>
> - LLM只负责生成调用意图
> - 后端做严格校验和执行

---

## 🔥 加分点：错误处理

```
错误类型：
- LLM hallucination
- Tool失败
- 网络问题
```

👉 解决：

- retry
- fallback
- degrade（直接给答案）
